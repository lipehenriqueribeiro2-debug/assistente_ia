Crie a especificação de layout (UI spec) para um sistema operacional focado em teclado com estas regras:

VISUAL DECISIONS:
- Sentimento: Um núcleo pulsante e responsivo, como um coração orbital multifacetado.
- Objeto hero: O núcleo 3D (esfera hexagonal contida em casca vazada).
- Trabalho: Orquestrar.

STYLE LOGICS:
- Cor: O espaço é um vácuo negro monocromático. A interface atua como vidro translúcido escuro (rgba(30, 41, 59, 0.6) + backdrop-filter: blur(16px)). Toda a cor (Fagulha) vem de estados ativos que herdam a cor do núcleo 3D. Sem box-shadows.
- Tipo: Tipografia sem-serifa compactada (Inter, letter-spacing negativo) para editorial, separada por fronteiras rígidas de 1px (hairline #3e3e4d) da tipografia monoespaçada mecânica (IBM Plex Mono) usada para dados.
- Espacial: O núcleo é o centro gravitacional; sem barras fixas ou contêineres permanentes. Apenas painéis de vidro translúcido efêmeros orbitam o núcleo quando invocados.

SECTIONS:
- Fundo contínuo (AiCore Canvas).
- Barra de comando universal flutuante (tipo Raycast, invocável via atalho).
- Modais translúcidos efêmeros para execução de sub-agentes.

LAYOUT RULES:
- Zero barras laterais, zero navegação de topo, zero widgets estáticos.
- A barra de comando e modais aparecem sobre o núcleo em z-index elevado (Glassmorphism), respeitando limites geométricos rígidos.
